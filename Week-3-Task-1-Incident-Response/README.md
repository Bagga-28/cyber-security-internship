# Incident Response Lifecycle

Incident response is a structured process used to identify, manage, and recover from cybersecurity incidents.

## 1. Identification

The organization detects and confirms that a security incident has occurred.

### Activities

* Monitoring alerts
* Log analysis
* Threat detection

## 2. Containment

The affected systems are isolated to prevent the attack from spreading.

### Activities

* Disconnect compromised devices
* Block malicious IP addresses
* Restrict access

## 3. Eradication

The root cause of the incident is removed.

### Activities

* Remove malware
* Patch vulnerabilities
* Delete malicious files

## 4. Recovery

Systems are restored to normal operation.

### Activities

* Restore backups
* Verify system integrity
* Monitor for recurring threats

## 5. Lessons Learned

Organizations analyze the incident and improve defenses.

### Activities

* Document findings
* Update policies
* Improve security controls

# Case Study: WannaCry Ransomware Attack (2017)

## Overview

WannaCry was a global ransomware attack that affected hundreds of thousands of computers across more than 150 countries.

## Attack Method

* Exploited a vulnerability in Microsoft Windows.
* Spread rapidly across networks.
* Encrypted user files and demanded ransom payments.

## Impact

* Hospitals, businesses, and government organizations were affected.
* Significant operational disruption occurred worldwide.

## Lessons Learned

* Importance of timely software updates.
* Need for regular backups.
* Importance of network segmentation.

# Hypothetical Incident Response Plan

## Scenario

An employee clicks a phishing email attachment that installs malware.

### Step 1: Identification

* Security team receives alerts from antivirus software.
* Suspicious activity is confirmed.

### Step 2: Containment

* Disconnect affected device from the network.
* Disable compromised accounts.

### Step 3: Eradication

* Remove malware.
* Scan all affected systems.
* Apply security patches.

### Step 4: Recovery

* Restore clean files from backups.
* Reconnect systems to the network.
* Monitor for unusual activity.

### Step 5: Lessons Learned

* Conduct employee awareness training.
* Strengthen email security controls.

# Security Best Practices Checklist

## System Security

* Keep software updated.
* Apply security patches regularly.
* Use antivirus and endpoint protection.

## Authentication Security

* Use strong passwords.
* Enable Multi-Factor Authentication (MFA).
* Avoid password reuse.

## Network Security

* Use firewalls.
* Monitor network traffic.
* Segment critical systems.

## Data Protection

* Encrypt sensitive data.
* Perform regular backups.
* Test backup recovery procedures.

## User Awareness

* Conduct cybersecurity training.
* Identify phishing attempts.
* Follow organizational security policies.
